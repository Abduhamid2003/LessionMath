## 1. Общие сведения

### 1.1. Основание для разработки

Веб-платформа **Calculus Visual** (calculus-platform) реализует математические модули, обучение, чат и роли пользователей. Для студентов и преподавателей требуется **нативное мобильное приложение**, которое:

- использует **тот же backend** (REST + WebSocket);
- обеспечивает удобный доступ с телефона и планшета;
- поддерживает push-уведомления, офлайн-просмотр уроков и быстрый вход.

### 1.2. Назначение мобильного приложения

Мобильный клиент предназначен для:

| Задача | Описание |
|--------|----------|
| Учёба «на ходу» | Уроки, тесты, калькулятор, графики |
| Коммуникация | Чат класса и личные сообщения |
| Уведомления | Push о новых уроках, тестах, сообщениях |
| Визуализация | Интерактивные графики с жестами (масштаб, панорама) |
| Роль преподавателя | Упрощённое управление классом и публикация контента |

**Администрирование** (полная панель admin) в мобильной v1.0 — **ограниченно** (просмотр статистики, смена роли — опционально); основной admin-функционал остаётся в веб-версии.

### 1.3. Целевая аудитория

Та же, что у веб-платформы: студенты, преподаватели, самостоятельные учащиеся. Приоритет MVP — **студент** и **преподаватель**.

### 1.4. Цели проекта (мобильные)

1. Дать **≥ 90%** ключевого студенческого функционала веба в приложении.
2. Сократить время доступа к калькулятору и графикам до **≤ 3 тапов** с главного экрана.
3. Обеспечить **push-уведомления** о учебных событиях и чате.
4. Сохранить **единую учётную запись** с веб (JWT, те же роли).
5. Подготовить публикацию в **Google Play** и **App Store**.

### 1.5. Границы проекта

**Входит в scope (v1.0):**

- Android и iOS из одной кодовой базы (кроссплатформа).
- Интеграция с существующим API calculus-platform.
- Математические модули через API (вычисления на сервере).
- Уроки, тесты, чат, профиль, уведомления.

**Не входит в scope v1.0:**

- Локальный SymPy на устройстве (все расчёты — через backend).
- Видеозвонки, голосовые сообщения.
- Полная замена веб-панели администратора.
- Платежи, подписки, реклама.
- Планшетная версия с отдельным UI (достаточно адаптивной вёрстки).
- Wear OS / watchOS.

## 2. Архитектура решения

### 2.1. Общая схема

```
┌──────────────────────────────────────────────────────────────┐
│           Calculus Visual Mobile (iOS / Android)             │
│  UI · Navigation · State · Secure Storage · Push · Charts    │
└────────────────────────────┬─────────────────────────────────┘
                             │ HTTPS (REST) + WSS
┌────────────────────────────▼─────────────────────────────────┐
│              calculus-platform (FastAPI backend)             │
│  /api/auth · /api/math · /api/lessons · /api/chat · ...      │
└────────────────────────────┬─────────────────────────────────┘
                             │
                    PostgreSQL / SQLite
```

### 2.2. Рекомендуемый технологический стек

| Слой | Рекомендация | Альтернатива |
|------|--------------|--------------|
| Кроссплатформа | **Flutter** 3.x (Dart) | React Native + TypeScript |
| HTTP-клиент | dio / http | axios (RN) |
| Состояние | Riverpod или Bloc | Redux Toolkit |
| Локальное хранилище | flutter_secure_storage (токен), Hive/Isar (кэш) | MMKV + Keychain |
| Графики | fl_chart или syncfusion_flutter_charts | react-native-chart-kit |
| Формулы (LaTeX) | flutter_math_fork / webview + KaTeX | react-native-katex |
| WebSocket чат | web_socket_channel | native WS |
| Push | Firebase Cloud Messaging (FCM) + APNs | — |
| i18n | flutter_localizations + ARB (ru, tg, en) | i18next |
| CI/CD | Codemagic / GitHub Actions + Fastlane | — |

**Обоснование Flutter:** единый UI для Android/iOS, хорошая работа с графиками и локализацией, быстрый MVP.

### 2.3. Конфигурация окружений

| Окружение | Base URL API | Назначение |
|-----------|--------------|------------|
| dev | `http://10.0.2.2:8000` (Android emulator) / `http://localhost:8000` | Разработка |
| staging | `https://api-staging.example.com` | Тестирование |
| prod | `https://api.calculus.example.com` | Продакшен |

Параметры в `.env` / `flutter_dotenv` / build flavors: `API_BASE_URL`, `WS_BASE_URL`.

### 2.4. Аутентификация в мобильном клиенте

| Требование | Реализация |
|------------|------------|
| Вход | `POST /api/auth/login` (username, password) |
| Регистрация | `POST /api/auth/register` |
| Токен | JWT в **Secure Storage** (Keychain / EncryptedSharedPreferences) |
| Обновление сессии | При 401 — экран входа, очистка токена |
| Заголовок | `Authorization: Bearer <token>` на все защищённые запросы |
| Биометрия (v1.1) | Face ID / Touch ID / отпечаток — разблокировка сохранённого токена |
| Сброс пароля | Deep link `calculus://reset-password?token=...` → экран смены пароля |

## 4. Ролевая модель

Идентична веб-платформе ([TZ.md §3](TZ.md)). Мобильное приложение читает `user.role` из JWT / `GET /api/auth/me`.

### 4.1. Матрица доступа (мобильное приложение)

| Функция | Гость | Студент | Преподаватель | Админ |
|---------|-------|---------|---------------|-------|
| Просмотр математических модулей | ✓ | ✓ | ✓ | ✓ |
| Регистрация / вход | ✓ | — | — | — |
| Уроки (опубликованные) | ✓* | ✓ | ✓ | ✓ |
| Тесты | — | ✓ | ✓ | ✓ |
| Чат | — | ✓ | ✓ | ✓ |
| Push-уведомления | — | ✓ | ✓ | ✓ |
| Панель преподавателя (моб.) | — | — | ✓ | ✓ |
| Панель администратора (полная) | — | — | — | веб |

\*Гость: уроки только для чтения без сохранения прогресса.

### 4.2. Навигация по ролям

**Студент / гость (Bottom Navigation, 5 вкладок):**

| Вкладка | Экран |
|---------|--------|
| Главная | Дашборд, быстрые действия |
| Учёба | Уроки + тесты |
| Математика | Хаб: графики, производные, интегралы, пределы, калькулятор |
| Чат | Список комнат и диалогов |
| Профиль | Настройки, достижения, выход |

**Преподаватель:** те же вкладки + пункт «Преподаватель» в профиле или отдельная вкладка вместо «Учёба» (конфигурируемо).

## 7. Дизайн и UX

### 7.1. Визуальный стиль

- Наследовать палитру веб: primary `#6366f1` (indigo), акценты для графиков.
- Шрифты: системные (Roboto / SF Pro) + моноширинный для формул.

### 7.2. UX-принципы

1. **Thumb zone:** основные действия внизу экрана.
2. **Одна главная кнопка** на математическом экране («Вычислить»).
3. **Не блокировать UI** при запросе — skeleton / shimmer.
4. **Ошибки:** человекочитаемый текст + «Повторить».
5. **Пустые состояния:** иллюстрация + подсказка (нет чатов → «Найдите учителя через поиск»).

```
GET  /api/chat/rooms/{id}/messages
WS   /api/chat/ws/{id}?token=
GET  /api/notifications
POST /api/notifications/{id}/read
GET  /api/achievements/me
```

### 8.2. Формат ошибок

Обрабатывать JSON:

```json
{ "detail": "сообщение" }
```

или массив validation errors (HTTP 422). Показывать первое сообщение пользователю.

### 8.3. Требуемые доработки backend (для мобильного)

| ID | Эндпоинт / изменение | Назначение |
|----|----------------------|------------|
| API-M01 | `POST /api/auth/device-token` | Сохранение FCM/APNs token |
| API-M02 | `DELETE /api/auth/device-token` | Выход, удаление token |
| API-M03 | Сервис отправки push при notify_students / chat | FCM + APNs |
| API-M04 | `GET /api/config/mobile` | min_version, feature flags, store URLs |
| API-M05 | CORS / без CORS для mobile | Не требуется для нативного клиента |
| API-M06 | Опционально: `GET /api/math/plot.png` | Серверный рендер графика для share |

### 8.4. WebSocket (чат)

- URL: `wss://{host}/api/chat/ws/{room_id}?token={jwt}`
- Ping каждые 20 с: `{ "type": "ping" }`
- Reconnect: exponential backoff (1s, 2s, 4s … max 30s)
- При сворачивании приложения — отключение WS; при foreground — reconnect

## 9. Офлайн-режим и кэширование

| Данные | Стратегия | Срок хранения |
|--------|-----------|---------------|
| JWT | Secure Storage | До logout |
| Уроки (текст) | SQLite/Hive | 7 дней, max 20 MB |
| Список комнат чата | Кэш + refresh | Последний snapshot |
| Сообщения | Только online в v1.0 | — |
| Математика | Только online | — |
| Настройки (язык, тема) | Local + sync preferences | — |

При offline на математическом экране — banner «Нет сети» и блокировка кнопки вычисления.

## 15. Пользовательские сценарии (мобильные)

### UC-M01. Студент решает задачу в автобусе

1. Открывает приложение (биометрия / уже в сессии).
2. Вкладка «Математика» → «Производные».
3. Вводит `x**3-4*x`, нажимает «Вычислить».
4. Смотрит графики, pinches для увеличения.
5. Открывает bottom sheet «Шаги».

**Результат:** обучение без ноутбука.

---

### UC-M02. Push о новом уроке

1. Преподаватель публикует урок в веб или моб. панели.
2. Backend шлёт FCM студентам класса.
3. Студент tap на уведомление → `SCR-LESSON`.
4. Читает урок; при offline — из кэша, если был открыт ранее.

---

### UC-M03. Студент пишет учителю

1. Вкладка «Чат» → поиск «teacher».
2. Tap на контакт → `SCR-CHAT-ROOM`.
3. Пишет сообщение; учитель получает push.
4. Ответ приходит по WebSocket без перезагрузки.

---

### UC-M04. Преподаватель зачисляет студента

1. Профиль → «Преподаватель» → Классы.
2. Выбирает класс → «Добавить студента».
3. Вводит username `student`, подтверждает.
4. Студент видит чат класса в списке комнат.
