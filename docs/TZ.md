## 1. Общие сведения

### 1.1. Основание для разработки

Необходимость создания единой веб-платформы для изучения дифференциального и интегрального исчисления с визуализацией, интерактивными заданиями и инструментами сопровождения учебного процесса (преподаватель — студент).

### 1.2. Назначение системы

Система предназначена для:

- визуального исследования функций, производных, интегралов и пределов;
- прохождения структурированных уроков и тестов;
- организации учебных классов и коммуникации между участниками;
- администрирования контента и пользователей.

### 1.3. Целевая аудитория

| Группа | Потребности |
|--------|-------------|
| Студенты (школа, колледж, вуз) | Понятная визуализация, практика, тесты, обратная связь |
| Преподаватели | Создание уроков/тестов, классы, мониторинг, уведомления |
| Администраторы | Управление пользователями, контентом, статистикой |
| Самостоятельные учащиеся | Калькулятор и модули без привязки к классу |

### 1.4. Цели проекта

1. Снизить порог входа в темы производной и интеграла за счёт графиков и пошаговых решений.
2. Объединить вычислительный движок (SymPy) и учебный контент в одном интерфейсе.
3. Поддержать многоязычность (русский, таджикский, английский).
4. Обеспечить ролевую модель доступа и безопасную работу с пользовательскими выражениями.

### 1.5. Границы проекта (вне scope веб-версии)

- Нативные мобильные приложения (iOS/Android) — см. отдельное ТЗ: **[TZ-mobile.md](TZ-mobile.md)**.
- Docker-образы и оркестрация (по решению заказчика — отдельная задача).
- Видеоконференции.
- Платёжная система и коммерческие подписки.
- Полноценная LMS с расписанием, оценками за семестр, SCORM.

## 5. Модель данных

### 5.1. Сущности (ER-описание)

```
User ──┬──< ClassGroup (teacher)
       ├──< ClassEnrollment >── ClassGroup
       ├──< Result
       ├──< UserAchievement >── Achievement
       ├──< Notification
       ├──< ChatRoomMember >── ChatRoom ──< ChatMessage
       └──< CalculationHistory

Lesson ── author: User
Test ──< TestQuestion
Test ── author: User, optional class_id
```

### 5.2. User

| Поле | Тип | Описание |
|------|-----|----------|
| id | int PK | Идентификатор |
| email | string unique | Email |
| username | string unique | Логин |
| hashed_password | string | Bcrypt-хеш |
| full_name | string | Отображаемое имя |
| role | enum | admin / teacher / student |
| is_active | bool | Активен ли аккаунт |
| preferred_language | string | ru / tg / en |
| preferred_theme | string | light / dark |
| reset_token | string? | Токен сброса пароля |

### 5.3. Lesson

| Поле | Тип | Описание |
|------|-----|----------|
| title_ru, title_tg | string | Заголовки |
| content_ru, content_tg | text | Содержание |
| category | string | Категория |
| order_index | int | Порядок сортировки |
| is_published | bool | Опубликован |
| author_id | FK User | Автор |

### 5.4. Test / TestQuestion

- Test: заголовки RU/TG, описание, category, max_score, is_published, class_id (опционально), author_id.
- TestQuestion: текст, варианты ответов (JSON), correct_answer(s), баллы.

### 5.5. ClassGroup / ClassEnrollment

- ClassGroup: name, teacher_id.
- ClassEnrollment: class_id, student_id (связь многие-ко-многим студент↔класс).

### 5.6. Chat

| Сущность | Поля (ключевые) |
|----------|-----------------|
| ChatRoom | room_type (`class` / `direct`), class_id?, title |
| ChatRoomMember | room_id, user_id |
| ChatMessage | room_id, sender_id, content, created_at |
| UserPresence | user_id, last_seen, is_online |

### 5.7. Notification

- user_id, title, message, link (URL внутри приложения), is_read, created_at.

## 6. Интерфейсы API

### 6.1. Базовый URL

`/api/{модуль}/...`

Авторизация: `Authorization: Bearer <JWT>` (кроме публичных math-эндпоинтов и register/login).

### 6.2. Сводная таблица префиксов

| Префикс | Назначение |
|---------|------------|
| `/api/auth` | Регистрация, вход, профиль, сброс пароля |
| `/api/math` | Все математические операции |
| `/api/lessons` | Уроки |
| `/api/tests` | Тесты для студентов |
| `/api/teacher` | Панель преподавателя |
| `/api/admin` | Панель администратора |
| `/api/chat` | Чаты, контакты, WebSocket |
| `/api/notifications` | Уведомления |
| `/api/achievements` | Достижения |
| `/api/progress` | Прогресс |
| `/api/formulas` | История и избранное |

### 6.3. WebSocket

- **URL:** `ws(s)://host/api/chat/ws/{room_id}?token={JWT}`
- **Сообщения клиента:** `{ "type": "ping" }`, `{ "type": "message", "content": "..." }`
- **Сообщения сервера:** `{ "type": "message", ... }`, `{ "type": "presence", ... }`, `{ "type": "pong" }`

### 6.4. Коды ошибок

| Код | Ситуация |
|-----|----------|
| 400 | Невалидное выражение, пустые данные |
| 401 | Нет или неверный JWT |
| 403 | Недостаточно прав (роль, членство в комнате) |
| 404 | Ресурс не найден |
| 429 | Превышен rate limit |

## 7. Пользовательские сценарии (Use Cases)

### UC-01. Студент строит график производной

1. Открывает `/derivatives`.
2. Вводит `x**3 - 4*x`, нажимает «Производная».
3. Видит графики f и f′, LaTeX-формулу и шаги.
4. (Опционально) экспортирует PDF.

**Результат:** понимание связи функции и производной.

---

### UC-02. Преподаватель публикует урок

1. Входит как `teacher`.
2. Открывает `/teacher` → вкладка уроков.
3. Создаёт урок, заполняет RU/TG контент.
4. Ставит галочку «Опубликовать и уведомить студентов».
5. Сохраняет.

**Результат:** студенты класса получают уведомление; урок виден в `/learning`.

---

### UC-03. Студент пишет учителю

1. Входит как `student` (зачислен в класс).
2. Открывает `/chat`.
3. В поиске вводит имя учителя.
4. Нажимает на контакт → открывается личный чат.
5. Отправляет сообщение.

**Результат:** сообщение доставлено через WebSocket; комната появляется в списке.

---

### UC-04. Администратор деактивирует пользователя

1. Входит как `admin`.
2. `/admin` → список пользователей.
3. Отключает `is_active` для нарушителя.

**Результат:** пользователь не может выполнять защищённые операции.

## 9. Требования к безопасности

1. **Пароли:** только в виде bcrypt-хеша в БД.
2. **JWT:** подпись SECRET_KEY; срок жизни настраивается.
3. **Выражения:** whitelist-подход через SymPy; блокировка опасных подстрок; max 500 символов.
4. **Rate limiting:** защита math API от злоупотреблений.
5. **CORS:** ограничение origin в production.
6. **Загрузка файлов:** только изображения преподавателем; сохранение в `static/uploads/`.
7. **Чат:** проверка членства в комнате; проверка права на direct-сообщения.
